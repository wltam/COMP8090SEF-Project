**Task 1 - OOP-based application development**

## Introduction
The Service Record Management System (SRMS) is an application developed using the Python programming language. It is designed to assist small companies in managing clients and invoices by replacing manual record-keeping methods. The system allows for the creation of client profiles, the tracking of job contracts, the issuing of invoices, and the recording of payments. This project was developed to demonstrate the practical application of  Object-Oriented Programming (OOP) concepts, such as classes, encapsulation, and lists.

##	Background and Motivation
Currently, many small businesses still rely on physical notebooks or independent Microsoft Excel spreadsheets to store data. While spreadsheets are useful for basic calculations, they are often insufficient for managing relationships between different types of data. For example, client details might be stored in one file, while financial records are kept in another. This separation makes it difficult to track which invoices belong to which client. 

As business data grows, information is often duplicated or lost and tracking cross-referenced information (such as which invoices belong to which client) becomes increasingly difficult as the volume of data grows. This project aims to solve these issues by providing a structured, Java-based system to centralize the data.
